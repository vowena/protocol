# Support

## Use issues for

- Reproducible bugs
- Scoped feature proposals
- Documentation gaps in this repository
- CI or tooling problems tied to protocol development

## Before opening an issue

- Read [README.md](README.md) for architecture and setup context.
- Read [CONTRIBUTING.md](CONTRIBUTING.md) for the verification bar.
- Read [SECURITY.md](SECURITY.md) if the issue may be security-sensitive.

## Questions and design discussion

If discussions are enabled for this repository, use them for early design conversations and open-ended questions. Otherwise, open an issue with enough detail for maintainers to triage it properly.
